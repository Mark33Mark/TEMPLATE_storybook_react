## ❓ About  

This is my rework of Chromatic's Storybook React template.  The main change was I removed the Typescript, mostly as I see it as unnecessary for this exercise.

## 🚅 Quick start  

**Browse your stories!**  

    Run `npm run storybook` to see your component's stories at `http://localhost:6006`

## 🔎 What's inside?  

A quick look at the top-level files and directories included with this template.

    .
    ├── .storybook
    ├── public
    ├── src
    ├── .gitignore
    ├── index.html
    ├── LICENSE
    ├── package.json
    ├── vite.config.js
    └── README.md

1.  **`.storybook`**: This directory contains Storybook's [configuration](https://storybook.js.org/docs/configure) files.

2.  **`public`**: This directory will contain the development and production build of the site.

3.  **`src`**: This directory will contain all of the code related to what you will see on your application.

4.  **`.gitignore`**: This file tells git which files it should not track or maintain during the development process of your project.

5.  **`index.html`**: This is the HTML page that is served when generating a development or production build.

6. **`LICENSE`**: The template is licensed under the MIT licence.

7. **`package.json`**: Standard manifest file for Node.js projects, which typically includes project specific metadata (such as the project's name, the author among other information). It's based on this file that npm will know which packages are necessary to the project.

8. **`vite.config.ts`**: This is the configuration file for [Vite](https://vitejs.dev/), a build tool that aims to provide a faster and leaner development experience for modern web projects.

9. **`README.md`**: A text file containing useful reference information about the project.
